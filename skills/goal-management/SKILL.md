# Skill: Goal Management

## Purpose
Manage and execute on user-defined goals through heartbeat-driven actions.

## Files
- `memory/GOALS.md` - Active goals with Definition of Done
- `memory/GOAL_LOG.md` - Progress log with alignment statements
- `HEARTBEAT.md` - System instructions for goal execution

## Usage

### Adding a Goal
1. User describes goal
2. Ask for Definition of Done (must be specific/measurable)
3. Add to GOALS.md with 🔴 status
4. Confirm goal is captured

### During Heartbeat
1. Read GOALS.md
2. Pick ONE goal (prioritize 🟡 in-progress, then 🔴 not started)
3. Take action toward that goal
4. Log to GOAL_LOG.md with timestamp and alignment statement
5. Update goal status in GOALS.md if needed
6. Report to user: what was done + how it aligns

### Goal Completion
When Definition of Done is met:
1. Update status to 🟢
2. Final log entry
3. Ask user if goal should be archived or removed

### Alignment Statement Format
"This action aligns with Goal #[N] by [specific explanation of contribution]"

## Rules
- ONE goal per heartbeat action (no multitasking)
- Every action must have explicit alignment statement
- Definition of Done must be clear before starting
- Log EVERY action toward a goal
- Never work on goals without logging
